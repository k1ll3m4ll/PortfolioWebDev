Zusammenfassung Webentwicklung

HTML als Auszeichnungssprache

HTML (HyperText Markup Language) ist eine textbasierte Auszeichnungssprache und dient zur Strukturierung und Darstellung von Inhalten auf Webseiten. Sie beschreibt die logische Struktur von Dokumenten mit Hilfe von Tags, die in Winkelklammern geschrieben werden (z. B. <p> für einen Absatz). HTML ist keine Programmiersprache, sondern ermöglicht das semantische Markieren von Inhalten wie Text, Bildern und Links.

Die DOM-Struktur

Das Document Object Model (DOM) ist eine baumartige Repräsentation einer HTML-Dokumentstruktur. Jeder Knoten im Baum stellt ein Element, Attribut oder Text dar. Der oberste Knoten ist das document, das Wurzelelement wird durch <html> repräsentiert. Untergeordnete Knoten sind z. B. <head> und <body>, die wiederum weitere Elemente wie <div>, <p>, <img> oder <script> enthalten können. Diese Struktur erlaubt es JavaScript, auf HTML-Elemente zuzugreifen und sie zu manipulieren.

Events in JavaScript

Events sind Ereignisse, die während der Interaktion mit einer Webseite auftreten. Sie spielen eine zentrale Rolle in der Interaktivität. Beispiele für Events sind:

  • click: Ausgelöst, wenn ein Benutzer auf ein Element klickt.
  • mouseover: Tritt auf, wenn der Mauszeiger über ein Element bewegt wird.
  • keydown: Wird ausgelöst, wenn eine Taste auf der Tastatur gedrückt wird.
  • submit: Aktiviert, wenn ein Formular übermittelt wird.

Events können durch Event-Listener behandelt werden, die bestimmten Code ausführen, sobald das Ereignis eintritt.

Compiler vs. Interpreter

Ein Compiler übersetzt den gesamten Quellcode einer Programmiersprache vor der Ausführung in Maschinensprache und generiert eine ausführbare Datei. Beispiele hierfür sind Sprachen wie C++ oder Java.

Ein Interpreter hingegen liest und führt den Quellcode Schritt für Schritt aus, ohne eine Zwischenübersetzung in Maschinensprache zu speichern. Beispiele hierfür sind JavaScript oder Python. Während ein Compiler in der Regel schneller ausgeführte Programme ermöglicht, erlaubt ein Interpreter einfachere und dynamischere Anpassungen während der Laufzeit.

JavaScript und DOM-Manipulation

Mit JavaScript lassen sich Inhalte und Attribute von DOM-Elementen verändern. Zum Beispiel kann durch folgende Codezeile der Text eines <p>-Elements aktualisiert werden: 

document.getElementById("example").textContent = "Neuer Textinhalt";

Auch das Hinzufügen von Event-Listenern zur Reaktion auf Benutzeraktionen ist ein zentraler Aspekt: 


document.querySelector("button").addEventListener("click", function() {
    alert("Button wurde geklickt!");
});
