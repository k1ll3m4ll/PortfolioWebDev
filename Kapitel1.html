Zusammenfassung Webentwicklung

Netzwerkkommunikation und Client-Server-Architektur

Die Vorlesung behandelt die Grundlagen der verteilten Systeme, insbesondere die Kommunikation zwischen Clients und Servern. Der Client ist typischerweise der schwächere Rechner (z.B. PC, Smartphone), während der Server leistungsfähiger ist und Dienste bereitstellt. Historisch entwickelten sich verschiedene Netzwerkarchitekturen von DECNET und IBM SNA in den 1970er Jahren über LAN (Ethernet) in den 1980ern bis zum modernen Internet (IP) ab den 1990ern.

Serverseitige Technologien

Serverseitige Technologien ermöglichen die dynamische Generierung von Webinhalten. Wichtige Konzepte sind:

  • CGI (Common Gateway Interface): Eine veraltete Technologie (1994-1995), die Sicherheits- und Performance-Probleme aufweist.
  • PHP: Eine Skriptsprache, die in HTML eingebettet wird und häufig mit Datenbanken wie MySQL kombiniert wird.
  • Java Servlets & JSP: Java-basierte Technologien für serverseitige Verarbeitung.
  • Node.js: Eine Laufzeitumgebung für JavaScript, die serverseitige Programmierung ermöglicht.

Java Serialisierung und Streams

Die Serialisierung von Objekten in Java ermöglicht das Speichern des Objektzustands. Ein Objektgraph wird dabei vollständig gespeichert, inklusive aller referenzierten Objekte. Für die Serialisierung muss die Klasse das Serializable-Interface implementieren. Transiente Variablen werden nicht serialisiert.

Java verwendet Streams für Ein- und Ausgabeoperationen. Wichtige Klassen sind:

  • FileInputStream/FileOutputStream für Dateizugriffe
  • ObjectInputStream/ObjectOutputStream für die Serialisierung
  • BufferedReader/BufferedWriter für gepufferte Ein-/Ausgabe

Dateiverarbeitung in Java

Die Klasse java.io.File repräsentiert Dateien und Verzeichnisse. Wichtige Methoden sind:

  • mkdir() - Erstellt ein Verzeichnis
  • list() - Listet Verzeichnisinhalte auf
  • delete() - Löscht eine Datei
  • exists() - Prüft Existenz

Für Textdateien werden FileReader und FileWriter verwendet, während für Binärdaten DataInputStream und DataOutputStream zum Einsatz kommen.

Netzwerkkomponenten

Netzwerkkomponenten wie Bridges und Switches spielen eine wichtige Rolle in verteilten Systemen:

  • Bridges: Verbinden Netzwerksegmente und filtern Frames basierend auf MAC-Adressen
  • Switches: Sind Multiport-Bridges, die Kollisionsdomänen eliminieren und parallele Verbindungen ermöglichen

Switches arbeiten hardwarebasiert und verwenden Store-and-Forward oder Cut-Through Switching für effiziente Datenübertragung.

Clientseitige Technologien

Neben serverseitigen Technologien gibt es wichtige clientseitige Ansätze:

  • JavaScript und Frameworks: Für dynamische Webinhalte
  • Java Applets: Veraltet und aus Sicherheitsgründen nicht mehr empfohlen
  • Native Apps: Swift für iOS, Java für Android
  • Hybride Apps: Kombinieren Web- und Native-Technologien
