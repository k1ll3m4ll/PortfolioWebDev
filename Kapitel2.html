Netzwerkprogrammierung mit Sockets

Die Vorlesung behandelt die Grundlagen der Netzwerkprogrammierung in Java, insbesondere die Verwendung von Sockets für die Client-Server-Kommunikation. Ein Socket repräsentiert eine Verbindung zwischen zwei Anwendungen, die auf verschiedenen Maschinen laufen können. Für die Verbindung benötigt der Client die IP-Adresse und Portnummer des Servers.

TCP/IP und Ports

TCP/IP ist das grundlegende Protokoll für die Netzwerkkommunikation. Ports sind 16-Bit-Zahlen, die bestimmten Anwendungen zugeordnet sind. Ports 0-1023 sind für bekannte Dienste reserviert (HTTP, FTP, etc.), während höhere Portnummern für eigene Anwendungen genutzt werden können. Ein Server kann mehrere Anwendungen gleichzeitig betreiben, jede auf einem eigenen Port.

Client-Server-Architektur

Der Server erstellt einen ServerSocket auf einem bestimmten Port und wartet auf Client-Anfragen. Bei einer Anfrage akzeptiert der Server die Verbindung und erstellt einen neuen Socket für die Kommunikation mit diesem Client. Der Client verbindet sich mit dem Server über dessen IP und Port.

Datenübertragung mit Streams

Die Kommunikation erfolgt über Input- und OutputStreams. Für Textdaten werden BufferedReader (zum Lesen) und PrintWriter (zum Schreiben) verwendet. Diese werden mit den Low-Level-Streams des Sockets verbunden. Der InputStreamReader fungiert als Brücke zwischen Byte- und Zeichenströmen.

Multithreading in Java

Threads ermöglichen parallele Ausführungsstränge in einer Anwendung. Jeder Thread hat seinen eigenen Aufruf-Stack. Ein Thread benötigt ein Runnable-Objekt, das die auszuführende Arbeit (run()-Methode) definiert. Threads können verschiedene Zustände haben: neu, lauffähig, laufend oder blockiert.

Thread-Scheduler und Synchronisation

Der Thread-Scheduler entscheidet, welcher Thread ausgeführt wird. Es gibt keine Garantien für die Ausführungsreihenfolge. Thread.sleep() kann verwendet werden, um die Ausführung zu pausieren. Bei gemeinsam genutzten Ressourcen kann es zu Race Conditions kommen, die durch Synchronisation gelöst werden müssen.

Praktisches Beispiel: Chat-Anwendung

Als praktisches Beispiel wird eine Chat-Anwendung vorgestellt. Der Server empfängt Nachrichten von Clients und verteilt sie an alle verbundenen Clients. Jeder Client benötigt einen eigenen Thread für das Empfangen von Nachrichten, während der Hauptthread für das Senden zuständig ist.
